Pos Cache

Enable a cache on pos models for a lower POS loading time.

Features.
  • Allow to configure witch models will be cached.
  • Creates a cache per POS config.
  • Drastically lowers the time it takes to load a POS session with a lot of model items.
  • Cache gets automatically updated when users edit or create new records belongs to configured catched models.
  • A daily cron rebuild all cache to ensure start new day with fresh catched data.
Allow to configure witch models will be catched.

To start to use this module, you have to configure the models that will be cached, for this there is a field called "Cache models" in the pos config, only you have to add some models and those will be cached at pos session start.

Drastically lowers the time it takes to load a POS session with a lot of model items.

Tests with 150000 customers and 7000 products. The load without the module takes approximately 3 minutes. With the installed module it is reduced to 14 seconds.

Cache gets automatically updated when users edit or create new records belongs to configured catched models.

For these updates to reach the POS, the screen must be refreshed, which is now fast or at the next session opening.

A daily cron rebuild all cache to ensure start new day with fresh catched data.

It is responsible for records that have been created directly by BBDD or others not coupled.
